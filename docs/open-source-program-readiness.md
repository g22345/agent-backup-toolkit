# Open-source program readiness

Checked: 2026-07-17.

Official references:

- [Codex for Open Source](https://developers.openai.com/community/codex-for-oss)
- [Application form](https://openai.com/form/codex-for-oss/)
- [Program terms](https://learn.chatgpt.com/docs/codex-for-oss-terms)

The program considers active public open-source projects and asks whether the
applicant is a core or primary maintainer. Repository usage/adoption, ecosystem
importance, and active maintenance may be considered. Selection is discretionary
and is not guaranteed.

## Evidence map

| Criterion | Current evidence | Status |
| --- | --- | --- |
| Public open-source repository | Apache-2.0 license and public-ready tree | Not public yet |
| Core/primary maintainer | Intended maintainer recorded in design spec and Git history | Locally evidenced |
| Active maintenance | Dated commits, changelog, roadmap, CI/test plan | Locally evidenced |
| Useful project scope | Reproducible encrypted backup/verify/restore CLI for agent workspaces | Implemented locally |
| Usage/adoption | `ADOPTERS.md` contains no claims | No public evidence yet |
| Ecosystem importance | Codex-first preset and generic agent-workspace design | Rationale only; not adoption evidence |

## Honest pre-application gate

- public repository reviewed for private paths, history, credentials, and foreign provenance;
- CI green on macOS/Linux and supported Python versions;
- real `age` synthetic drill passes;
- at least one tagged alpha release with accurate documentation;
- genuine external tester feedback or adoption evidence, if obtained;
- application answers shown to the maintainer and verified against live public evidence;
- no claim of selection probability, security audit, production reliability, or adoption without proof.
